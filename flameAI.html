FlameAI

Project Description:

Flame_AI is an AI-powered information assistant designed to provide students, faculty, and visitors with real-time, accurate, and structured responses about Claremont Graduate University (CGU). Built using Streamlit, OpenAI’s GPT-4o, and Tavily Search, the project enables users to retrieve up-to-date information on course registration, student events, career opportunities, and administrative resources with a conversational AI interface.

Key features:

  • Conversational AI Agent – Developed an LLM-powered chatbot using LangChain to engage users in dynamic conversations
  • Live Web Search Integration – Implemented Tavily Search API to provide the latest CGU-related information, ensuring responses are always relevant.
  • Custom Chat Prompt Engineering – Designed a structured AI reasoning framework to deliver context-aware and structured responses.
  • User-Friendly Web Application – Built with Streamlit, offering an intuitive interface for users to interact seamlessly.

Implementing Real-Time Web Search for Up-to-Date Responses

Enhances the AI assistant’s ability to fetch and provide current, validated information.


                            from langchain_community.tools import TavilySearchResults
                            from langchain_core.tools import Tool
                            
                            # Initialize search tool
                            tavily_search = TavilySearchResults(
                                max_results=3, search_depth="advanced", include_answer=True,
                                api_key=os.getenv("TAVILY_API_KEY")
                            )
                            
                            # Create tool for CGU-specific searches
                            tools = [
                                Tool(
                                    name="SearchCGU",
                                    func=tavily_search.invoke,
                                    description="Fetches real-time CGU-related information: events, courses, tuition, career opportunities."
                                )
                            ]

Project 2:

AI-Powered Career Evaluator (ACE)

Project Description:

ACE is an intelligent agentic chatbot designed to support university students looking to break into the tech industry, through personalized resume reviews, interview prep, and job-market insights considering the rapid growth in the tech industry. The system uses Retrieval-Augmented Generation (RAG) and LangChain agents to deliver real-time, user support.

Key features:

  • LLM agent framework using LangChain + OpenAI GPT-4o & GPT-4omini
  • Dynamic prompt engineering for role-specific support
  • Integrated PGVector for semantic search (in progress)
  • Streamlit-based UX with LangSmith feedback tracing

Future Improvements & Contraints:

  • Redesign the user interface to be more intuitive, visually appealing, and consistent with CGU’s digital branding, including mobile responsiveness. As part of this, implement profile-based personalization to tailor content and recommendations based on students’ specialization, skill set, and career objectives.
  • Develop a student dashboard feature for tracking applications, saved opportunities, upcoming appointments, and relevant career resources.
  • Enable appointment scheduling with the Career & Professional Development Office via Handshake, allowing students to book sessions directly through the chatbot. This can also be used as a fallback feature in the case that the chatbot cannot provide specific information

Project 3:

LATransitRec: Olympics2028

Project Description:

The LATransitRec is a transit recommendation application designed to help visitors efficiently navigate Los Angeles during the 2028 Olympic Games. The app provides real-time traffic updates, public transit schedules, ride-sharing integrations (Uber, Lyft), and eco-friendly transport options, ensuring seamless, cost-effective, and sustainable travel experiences.
Developed by a tight-knit team of five graduate students following an Agile methodology, the project was structured into iterative sprints, focusing on feature development, user experience, and API integrations. As the appointed Scrum Master, I facilitated sprint planning, daily stand-ups, and backlog management to ensure efficient collaboration and timely feature delivery.

Key features:

  • Transit Recommendations – Personalized travel suggestions based on location, budget, real-time conditions, and sustainability preferences.
  • Multi-Modal Transportation Integration – Seamlessly integrates public transit, ride-sharing (Uber, Lyft), electric scooters, and bike-sharing
  • Live Traffic & Transit Updates – Real-time information from Google Maps API, LA Metro API, and traffic monitoring systems to prevent travel delays
  • User-Centered Design – Built with interactive mapping, filtering options, and accessibility features for intuitive trip planning.
  • Agile Development Approach – Developed in a collaborative team using Scrum, ensuring fast iterations and continuous user feedback incorporation.

Future Improvements & Contraints:

  • Developing AI-Powered integration for an improved user experience
  • Using GTFS data to replace API calls (in consideration) due to cost and restrictions from private APIs like Uber & Lift
  • More unit and integration testing
  • Team members also working on other class projects, needed flexibility with agile methods
